# Планируемые фичи

- Добавить возможность разных стратегий генераций треков
  - Включать и выключать каждую стратегию можно чекбоксом в отдельном блоке UI
  - Сейчас есть только одна стратегия через топ треки рекомендованных артистов по сиду который берётся из нашего списка Tracks
  - Новые стратегии
    - Фильтр на explicit content, в last.fm вроде нет такого, тогда надо смотреть при поиске треков в Spotify
    - Использовать случайный track.getSimilar по сиду
    - При добавлении рекомендованного трека любой стратегии по сиду, перед добавлением рекомендованного трека в очередь, добавлять сначала сам сид
      - Сделать числовой параметр: один раз на сколько сидов осуществлять такое добавление
      - Такие параметры размещать в одной строке после label чекбокса
    - Добавлять случайный трек из библиотеки аутентифицированного last.fm пользователя.
      - Надо при этом вычислять количество доступных страниц (`<recenttracks user="RJ" page="1" perPage="10" totalPages="3019">`) 
      - Наверное для запросов user.getRecentTracks лучше брать лимит чем меньше тем лучше (проверить), чтобы не грузить сеть и API. В целом perPage="10" норм.
    - Добавлять случайный loved трек из библиотеки пользователя тем же образом
      - Проверить если задавать limit и pages будет ли ответ приходить в таком же виде как recenttracks, если нет, то придумать как можно вычислить totalPages
    - Брать случайный трек из библиотеки пользователя в качестве сида, который будет отдаваться рандомным образом на одну из включенных стратегий принимающих сид (track.getSimilar, artist.getSimilar, ...)
    - Брать случайный loved трек в качестве сида, ...
    - Генерировать трек того же артиста по сиду.
      - Добавить числовой параметр: раз на сколько сидов осуществлять такое добавление.

- Вынести все api key и secret в .env, можно использовать gem dotenv для работы с ним.
